### **Summary:**

The **WebReg App** is a mobile application built for UCSD students to facilitate schedule sharing and collaboration. This app addresses the limitations of WebReg by enabling students to create, share, and compare schedules seamlessly, enhancing academic collaboration. 

The project was developed collaboratively with a team and focuses on user-centric features like real-time chat, friend management, and profile customization. The app is designed to improve the course planning process for UCSD students by providing intuitive and interactive tools.

---

### **My Role:**

As a **Frontend Developer**, I contributed to the project by:
- **Frontend Development**: Designed and implemented responsive, user-friendly interfaces using **React Native**.  
- **Core Feature Development**: Built key features such as schedule creation, profile management, and friend search functionality.  
- **Real-Time Communication**: Integrated **Socket.IO** to enable real-time chat functionality for improved collaboration.  
- **Team Collaboration**: Worked with a team of six, ensuring seamless integration with the Firebase backend.  
- **Performance Optimization**: Focused on enhancing user experience through efficient UI and feature implementations.  

---

### **Core Features:**

#### **Schedule Management**  
- **Create Schedules**: Add courses and build personalized schedules.  
- **Share Schedules**: Make schedules publicly accessible for collaborative course planning.  

#### **Friends Integration**  
- **Add & Manage Friends**: View and compare schedules with friends.  
- **Friend Requests**: Send, receive, and manage friend requests directly in the app.  

#### **Profile Customization**  
- **Set Up Profiles**: Create profiles with custom photos and hashtags for easy discovery.  
- **Edit Profiles**: Update profile details and manage hashtags for better visibility.  

---

### **Tech Stack**

#### **Frontend**:
- **React Native**: Developed an interactive and mobile-optimized interface.

#### **Backend**:
- **Firebase**: Utilized for authentication, real-time database management, and backend logic.

#### **Database**:
- **Firebase Realtime Database**: Stored user profiles, schedules, and friend connections.
